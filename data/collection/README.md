# AllerFit Data Collection

데이터 수집은 이 폴더의 CSV 파일 2개만 관리한다.

## 파일

```text
brands.csv
menu_data.csv
```

## brands.csv

브랜드별 수집 가능 여부와 우선순위를 관리한다.

주요 상태값:

```text
확인 전
공식 정보 있음
메뉴별 확인 필요
정보 없음
보류
```

## menu_data.csv

메뉴별 알레르기 유발 성분과 원산지 정보를 한 줄로 관리한다.

알레르기 성분은 DB의 `allergens.code` 기준으로 적고, 여러 개는 세미콜론으로 구분한다.

예시:

```text
milk;soybean;wheat
```

원산지는 초기 수집 단계에서는 쪼개지 않고 사람이 읽기 쉬운 원문 형태로 적는다.

예시:

```text
닭고기: 국내산; 돼지고기: 국내산; 치즈: 미국산
```

## 메뉴 데이터 상태값

```text
수집 전
수집 완료
검수 완료
정보 없음
보류
```

앱과 DB seed에는 기본적으로 `검수 완료` 데이터만 사용한다.

## 수집 원칙

- 공식 출처를 우선한다.
- 추정해서 입력하지 않는다.
- 원문 또는 근거 문구를 `raw_text`에 남긴다.
- 확인일은 `yyyy-mm-dd` 형식으로 적는다.
- 알레르기 정보가 없으면 빈칸으로 추정하지 말고 `정보 없음` 상태로 둔다.
